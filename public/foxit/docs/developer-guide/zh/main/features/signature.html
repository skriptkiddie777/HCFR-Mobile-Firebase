# 数字签名

# 7.3及7.4差异

在7.3版本中，foxit pdf sdk for web首次对签名功能做了支持。从7.4开始，foxit pdf sdk for web采用了新的签名实现方案，更易于开发使用。 在介绍7.3与7.4差异前，简单介绍标准的签名及验证（filter为Adobe.PPKLite，subfilter为adbe.pkcs7.sha1）的步骤。

note

此处标准指的是符合PDF Reference 1.7规范，不同app之间可以认同其他app的数字签名。

# 添加签名及签名验证步骤

  • 添加签名

    首先，生成包含签名覆盖范围信息（签名域字典的byteRange）的文档文件流。

    其次，对签名覆盖范围的文件流计算相应的摘要。

    再次，使用数字证书对摘要进行签名。

    最后，将签名的数据写入到步骤已生成的文档文件流中。

  • 验证签名

    首先，获取文档原始（未修改的）文件流，签名覆盖范围信息（签名域字典的byteRange）及签名数据。

    其次，对签名覆盖范围的文件流计算相应的摘要。

    最后，对签名数据及摘要进行验证，得出验证结果（包括文档是否被修改、签名者信息是否有效，时间戳是否过期等）。

# 7.3版本

在7.3版本中，开发者需要向Foxit PDF SDK for Web注册三个http接口（分别用于计算摘要、对摘要签名、验证）。换言之，开发者必须有一个后端来配置Foxit PDF SDK for Web实现数字签名的相关功能。如果开发者没有后端开发能力，那么就无法使用Foxit PDF SDK for Web的数字签名功能。

# 7.4版本

在7.4中，Foxit PDF SDK for Web设计了新的数字签名方案，开发者既可以通过后端形式实现计算摘要、对摘要签名、验证的算法，也可以使用js实现。

# 关于Foxit Demo

在我们的demo中，使用了一个后端来实现上述三个算法，其中： 摘要：使用openssl实现的sha1算法 签名：使用openssl实现的pkcs7证书签名算法 验证：使用openssl实现的pkcs7证书验证算法 具体接口地址如下： 中国区： http://webviewer-demo.foxitsoftware.cn/signature/digest_and_sign http://webviewer-demo.foxitsoftware.cn/signature/verify

海外区: http://webviewer-demo.foxitsoftware.com/signature/digest_and_sign http://webviewer-demo.foxitsoftware.com/signature/verify https://webviewer-demo.foxitsoftware.com/signature/digest_and_sign https://webviewer-demo.foxitsoftware.com/signature/verify

开发者如需使用以上接口来体验数字签名功能，可添加如下代码：

//origin 变量指要访问的域名，如:
//http://webviewer-demo.foxitsoftware.cn; https://webviewer-demo.foxitsoftware.com
//signature handlers
var requestData = function(type, url, responseType, body){
    return new Promise(function(res, rej){
        var xmlHttp = new XMLHttpRequest();
        xmlHttp.open(type, url)
        xmlHttp.responseType = responseType || 'arraybuffer';
        let formData = new FormData();
        if (body) {
            for (let key in body) {
                if (body[key] instanceof Blob) {
                    formData.append(key, body[key], key);
                } else {
                    formData.append(key, body[key]);
                }
            }
        }
        xmlHttp.onloadend = function(e) {
            let status = xmlHttp.status;
            if ((status >= 200 && status < 300) || status === 304) {
                res(xmlHttp.response);
            }else{
                rej(new Error('Sign server is not available.'));
            }
        };
        
        xmlHttp.send(body ? formData : null);
    });
}
//设置签名验证函数
pdfui.setVerifyHandler(function (signatureField, plainBuffer, signedData){
    return requestData('post', origin+'/signature/verify', 'text', {
        filter: signatureField.getFilter(),
        subfilter: signatureField.getSubfilter(),
        signer: signatureField.getSigner(),
        plainContent: new Blob([plainBuffer]),
        signedData: new Blob([signedData])
    });
});
//设置签名信息及签名函数, 此函数可调用多次以注册不同的签名算法及签名信息
pdfui.registerSignHandler({
    filter: 'Adobe.PPKLite',
    subfilter: 'adbe.pkcs7.sha1',
    flag: 0x100,
    distinguishName: 'e=support@foxitsoftware.cn',
    location: 'FZ',
    reason: 'Test',
    signer: 'web sdk',
    showTime: true,
    sign: function(setting, buffer) {
        return requestData('post', origin+'/signature/digest_and_sign', 'arraybuffer', {
            plain: new Blob([buffer])
        })
    }
});

# 与数字签名交互

界面上的签名及验证交互功能需要先调用 PDFUI.registerSignHandler(signerInfo) 及 PDFUI.setVerifyHandler(verifyFunction)接口设置签名及验证函数。

  • 添加一个新的签名域
  1. 点击界面中form 标签下的添加签名工具
  2. 在页面中画出一个签名区域
  3. 回退到手型工具状态
  4. 点击签名域，弹出签名设置窗口，配置好后点击OK按钮即可签名

Note WEB SDK会自动打开签名后的文档。

  • 已有签名域

    点击已有的签名域，弹出签名设置窗口，配置好后点击OK按钮即可签名

  • 验证签名

    手型工具状态下点击一个已签名过的签名域即可自动进行签名，并弹出窗口显示当前签名状态。

# 接口

# PDFUI.registerSignHandler(signerInfo)

接口PDFUI.registerSignHandler(signerInfo)用于注册签名句柄，其参数指定了使用的filter、subfilter、签名者以及默认的外观信息等。 开发者可多次调用此接口来注册不同的签名函数。 如：

pdfui.registerSignHandler({
    filter: 'Adobe.PPKLite',
    subfilter: 'adbe.pkcs7.sha1',
    flag: 0x100,
    distinguishName: 'e=example@foxitsoftware.cn',
    location: 'FZ',
    reason: 'Test',
    signer: 'web sdk',
    //sign handler, return a Promise with signed data;
    sign: (signInfo, buffer) => {
        //function getDigest() and sign() should be completed by developer.
        let digest = getDigest(buffer);
        let signData = sign(digest);
        return Promise.resolve(signData)
    }
})

# PDFUI.setVerifyHandler(verifyFunction)

接口PDFUI.setVerifyHandler(verifyFunction)用于设置签名的验证函数，当一个签名需要验证时，该回调函数将被调用。该回调函数返回签名的状态值state（整数）。请参阅 api 文档的 Signature_State 变量。 一个UIExtension仅需要调用一次PDFUI.setVerifyHandler(verifyFunction)。在回调函数中通过参数signatureField来判断使用何种验证算法。 如：

pdfui.setVerifyHandler((signatureField,plainBuffer,signedData) => {
    //function getDigest() and verify() should be completed by developer.
    let digest = getDigest(plainBuffer);
    let verifiedStatus = verify(signatureField.getFilter(),signatureField.getSubfilter(),signatureField.getSigner(),digest,signedData)
    return Promise.resolve(verifiedStatus)
});

# PDFDoc.sign(signInfo,digestSignHandler)

接口PDFDoc.sign(signInfo,digestSignHandler)用于对文档签名（需传入摘要、签名回调函数）。

/**
 * @returns {Blob} - File stream of signed document.
 */
const signResult= await pdfdoc.sign(signInfo,(setting, buffer) => {
    //function getSignData() should be completed by developer.
    return Promise.resolve(getSignData(setting,buffer))
});

# PDFDoc.verifySignature(verifyHandler)

接口PDFDoc.getOriginalStream()用于验证签名（需传入验证回调函数）。

/**
 * @returns {number} - Signature state.
 */
var result = await singedPDF.verifySignature(pdfform.getField('Signature_0'), function verify(signatureField, plainBuffer, signedData, hasDataOutOfScope){
    //function verifySignData() should be completed by developer.
    let signInfo={
        byteRange:signatureField.getByteRange(),
        signer:signatureField.getSigner(),
        filter:signatureField.getFilter(),
        subfilter:signatureField.getSubfilter(),
    }
    return Promise.resolve(verifySignData(signInfo,buffer))
});

# PDFSignature 类

通过PDFSignature 类可获取签名域的相关信息：

  • PDFSignature.isSigned(): 是否被签名过
  • PDFSignature.getByteRange(): 获取签名的覆盖范围
  • PDFSignature.getSigner(): 获取签名所有者
  • PDFSignature.getFilter(): 获取签名的filter
  • PDFSignature.getSubfilter(): 获取签名的subfilter
