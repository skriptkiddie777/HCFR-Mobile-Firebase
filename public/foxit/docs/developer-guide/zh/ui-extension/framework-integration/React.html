# FoxitPDFSDK 网页版集成示例 - React.js

本指南提供了两个示例。 一个介绍了如何在FoxitPDFSDK for Web中快速运行react.js的样板示例，另一个介绍了如何将FoxitPDFSDK for Web集成到使用WebPack和Babel创建的现有React应用中。

# 快速运行React.js样板示例

备注:FoxitPDFSDK for Web 根目录以下简称根目录或root.

FoxitPDFSDK for Web提供了使用WebPack和Babel创建的React应用的样板项目示例。 该示例可以在root/integrations/中找到。

# 样板示例目录结构说明

├─app/
│  ├─components/
│  │  └─PDFViewer/
│  ├─containers/
│  │  └─App/
│  ├─foxit-lib/
│  │    ├─...
│  ├─app.js
│  ├─index.html
│  ├─preload.js
│  └─license-key.js
├─development/
│  ├─webpack/
│  │    ├─...
│  └─setup.js
├─package.json
├─babel.config.js

# 几个关键文件/目录说明

File/Folder Description
app/ 包含这个应用的所有的JS、CSS等文件
development/ 包含所有应用开发环境的打包、应用初始化等自动化脚本
package.json 列举项目依赖、版本号等信息
app/components/PDFViewer/ 包含用于初始化 WebPDFSDK 的组件
app/preload.js 预加载SDK 底层资源的入口
app/app.js 应用入口

# 前提条件

  • Nodejs and npm
  • FoxitPDFSDK for Web

# 开始

在SDK的 root/integratons/react.js/ 下，开启命令行终端，执行以下命令来初始化项目：

npm run setup

此setup命令包含两部分操作：

  • npm install 安装相关依赖。to install dependencies.
  • npm run update-sdk
    • 从SDK根目录复制lib到root/integratons/react.js/app/目录下，并自动把lib重命名为foxit-lib。
    • 复制root/examples/license-key.js文件到root/integratons/react.js/app/目录下。

# 运行示例

运行下面命令启动开发服务，

npm start

现在您可以启动该应用程序了。打开浏览器，在地址栏输入<http://127.0.0.1:9102/> 来加载示例。

# 构建

npm run build

构建的产物生成到 root/integratons/react.js/dist 目录。

# 验证构建效果

这里可以简单使用 https://www.npmjs.com/package/http-server 启动 http 服务来查看效果：

cd ./dist && http-server -p 8080

# 将FoxitPDFSDK for web 集成到现有的React.js项目中

此集成过程假设您已经安装了由WebPack和Babel创建的React应用程序。

# 前提条件

  • Nodejs and npm
  • Reac.js created with WebPack and Babel
  • FoxitPDFSDK for Web

# Webpack 配置

让我们将您现有React项目的根文件夹和FoxitPDFSDK for Web分别称为ReactJS 和 SDK。

  1. 在“ReactJS/development/webpacK”文件夹中创建和配置以下3个文件:

    • webpack.base.js
    • webpack.dev.js
    • webpack.prod.js

    具体配置请参考 SDK/integrations/ response .js/development/webpack/中的对应文件。你也可以直接复制文件到ReactJS/development/webpacK。

  2. 在package.json中配置npm脚本

"script": {
"start": "webpack-dev-server --config development/webpack/webpack.dev.js",
"build": "webpack --config development/webpack/webpack.prod.js"
}

# 添加依赖项和入口点文件

  1. 在SDK中，复制lib 和 SDK/examples/license-key.js到ReacJS/app，并将lib名称改为 fox -lib。此外，为了正确引用字体库，还需要将SDK中的 external文件夹复制到ReactJS/app/foxit-lib/assets中。

  2. 在ReacJS中创建和配置以下文件:

    • the babel.config.js
    • the ../app/components/PDFViewer/index.js
    • the ../app/containers/App/index.js
    • the index.htm,app.js and preload.js inside ../app/

有关配置细节，请参考SDK中相应的文件。您还可以直接将这些文件复制到ReactJS中的对应文件夹中。

# 引用 Addons

如果你是把SDK 集成到您现有的工程中，您需要先了解一下本章节以了解如何引用插件。 我们将介绍三种引用插件的方式, 您只要根据需要选择选择其中一种就可以了。 关于插件的更多细节，请参考插件 .

# 1. 引用 Addon 碎片

碎片化引用是在7.2 及之前的版本中使用。这个方法无需任何配置，在 PDFViewer.vue 中构造 PDFUI 实例时按如下方式引用Addon：

this.pdfui = new UIExtension.PDFUI({
    addons: [
        the_path_to_foxit_lib + '/uix-addons/file-property/addon.info.json',
        the_path_to_foxit_lib + '/uix-addons/full-screen/addon.info.json',
        // .etc
    ],
    // other options
});

其中，the_path_to_foxit_lib 浏览器访问SDK lib目录的地址。

# 2. 模块化导入 Addon

  1. Install

    npm i -D @foxitsoftware/addon-loader

  2. 修改vue.config.js， 参考 /integrations/vue.js/vue.config.js

  3. 在PDFViewer.vue中导入各个 addon 的 addon.info.json

        import * as UIExtension from '../foxit-lib/UIExtension.full.js'
    import filePropertyAddon from '../foxit-lib/uix-addons/file-property/addon.info.json';
    import multiMediaAddon from '../foxit-lib/uix-addons/multi-media/addon.info.json';
    import passwordProtectAddon from '../foxit-lib/uix-addons/password-protect/addon.info.json';
    import redactionAddon from '../foxit-lib/uix-addons/redaction/addon.info.json';
    import pathObjectsAddon from '../foxit-lib/uix-addons/path-objects/addon.info.json';
    import printAddon from '../foxit-lib/uix-addons/print/addon.info.json';
    import fullScreenAddon from '../foxit-lib/uix-addons/full-screen/addon.info.json';
    import importFormAddon from '../foxit-lib/uix-addons/import-form/addon.info.json';
    import exportFormAddon from '../foxit-lib/uix-addons/export-form/addon.info.json';
    import undoRedoAddon from '../foxit-lib/uix-addons/undo-redo/addon.info.json';
    import textObjectAddon from '../foxit-lib/uix-addons/text-object/addon.info.json';

    同时构造PDFUI对象是这样传参：

    const libPath = '/foxit-lib/';
    this.pdfui = new UIExtension.PDFUI({
        addons: [
            filePropertyAddon,
            multiMediaAddon,
            passwordProtectAddon,
            redactionAddon,
            pathObjectsAddon,
            printAddon,
            fullScreenAddon,
            importFormAddon,
            exportFormAddon,
            undoRedoAddon
        ].concat(
            // 移动端不支持text-object addon
            UIExtension.PDFViewCtrl.DeviceInfo.isMobile
                ? []
                : textObjectAddon
        ),
        // other options
    });

# 3. 引用 allInOne.js

在foxit-lib/uix-addons/目录下有一个已经合并好的allInOne.js，在PDFViewer.vue中这样导入：

// ...
import * as UIExtension from '../foxit-lib/UIExtension.full.js';
import * as Addons from '../foxit-lib/uix-addons/allInOne.js';
// ...

在构造 PDFUI 实例时这样传参给PDFUI:

this.pdfui = new UIExtension.PDFUI({
    addons: UIExtension.PDFViewCtrl.DeviceInfo.isMobile
        ? Addons.filter(it => it.getName() !== 'textEditObject')
        : Addons,
    // other options
});

# 三种引用方式对比

引用方式 配置 增加的浏览器请求 可修改(国际化词条,addon.info.json中的配置==)
碎片化 无 n+ 支持修改
模块化 要配置gulp 0 支持修改，修改后需重新合并
allInOne 无 一个 不支持修改

# 运行

npm run start

现在一切都准备好了。打开浏览器，导航到http://127.0.0.1:9102/ 来启动应用程序。
