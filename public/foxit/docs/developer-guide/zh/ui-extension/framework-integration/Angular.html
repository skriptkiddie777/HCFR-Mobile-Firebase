# FoxitPDFSDK 网页版集成示例 - Angular

本指南提供了两个示例。一个示例介绍如何在FoxitPDFSDK for Web中快速运行Angular样板示例，另一个演示如何将FoxitPDFSDK for Web项目集成到您现有的Angular/cli应用程序中。

# 快速运行Angular样板示例

备注:FoxitPDFSDK for Web 根目录以下简称根目录或root.

FoxitPDFSDK for Web 为@angular/cli 应用程序提供了样板项目示例。该示例可以在root/integrations/中找到。

# 前提条件

  • Nodejs and npm
  • FoxitPDFSDK for Web

# 开始

在SDK的 root/integratons/angular.js/ 下，开启命令行终端，执行：

npm i

此步骤将执行以下操作：

  • 在当前目录创建一个node_modules文件夹并安装相关依赖。
  • 从SDK根目录复制lib目录到root/integrations/angular/src目录下，并自动把lib重命名为foxit-lib。

# 运行示例

在命令行终端上，，执行以下命令来启动Web应用程序：

npm start

现在您可以启动该应用程序了。打开浏览器，在地址栏输入<http://localhost:4200> 来加载示例。

# 将FoxitPDFSDK for web 集成到现有的Angular项目中

此集成过程假设您已安装应用程序@Angular/cli。

# 前提条件

  • Nodejs and npm
  • @angular/cli
  • FoxitPDFSDKforWeb

# 基础配置

让我们将您现有的Angular项目及FoxitPDFSDK for Web 的根目录简称为 AngularJS 和 SDK.

  1. 在SDK中找到 lib 文件夹，将其复制到 AngularJS/src/，然后将其重命名为 foxit-lib。 此外，要正确引用字体库，还需要将SDK中的 external 文件夹复制到 AngularJS/src/foxit-lib/assets。

在 AngularJS 中, 执行以下操作:

  1. 在angular.json中, update architect/build 选项 assets,styles 和 extractCss, 及architect/lint。 具体如下：

    {
  ...
  "build": {
    "assets": [
      ...,
      {
        "glob": "**/*",
        "input": "src/foxit-lib",
        "output": "/foxit-lib",
        "ignore": ["PDFViewCtrl.*", "UIExtension.*"]
      }
    ],
   "styles": [
       "src/foxit-lib/UIExtension.css",
       "src/styles.css"
     ],
     "extractCss": true,
     ...
  }
}
"lint": {
      "builder":...,
      "options": {
        "tsConfig": [
          //existing configuration can remain as they are
        ],
        "exclude": [
          // other dependencies you may have
          "src/foxit-lib/**/*.*"
        ]
      }
    },

  2. 在 tsconfig.app.json 中，添加 "src/foxit-lib/**/*.*" 到 exclude 列表.

    {
  ...,
  "exclude": [
    ...
    ...,
    ...,
    "src/foxit-lib/**/*.*"
  ]
}

# 创建组件

  1. 在 AngularJS 中, 启动命令行终端，运行：

    ng generate component PDFViewer

此步骤将在AngularJS/src/app下创建pdfviewer文件夹及相关组件文件。现在，您需要在AngularJS/src/app/下执行以下步骤：

  1. 将 license-key.js 放入 ../pdfviewer/. 您可以在 SDK/examples/找到许可证信息.

  2. 更新 ../pdfviewer/component.ts. 有关配置的详细信息，请参阅SDK中的对应文件

  3. 更新 ../component.html 来传递用于放置WebViewer容器的DOM元素

    <div>
  <app-foxitpdfviewer
    #pdfviewer
    class="foxit-pdf-viewer-container"
  ></app-foxitpdfviewer>
</div>

  4. 按您喜欢的样式更新 component.css。

    .foxit-pdf-viewer-container {
  display: block;
  margin: 0 auto;
  width: 1280px;
  height: 1024px;
}

# 引用Addons

如果你是把SDK 集成到您现有的工程中，您需要先了解一下本章节以了解如何引用插件。 我们将介绍三种引用插件的方式, 您只要根据需要选择选择其中一种就可以了。 关于插件的更多细节，请参考插件 .

# 1. 引用 Addon 碎片

碎片化引用是在7.2 及之前的版本中使用。这个方法无需任何配置，在 pdfviewer.component.ts 的onInit方法中构造 PDFUI 实例时按如下方式引用Addon：

this.pdfui = new UIExtension.PDFUI({
    addons: [
        the_path_to_foxit_lib + '/uix-addons/file-property/addon.info.json',
        the_path_to_foxit_lib + '/uix-addons/full-screen/addon.info.json',
        // .etc
    ],
    // other options
});

其中，the_path_to_foxit_lib 浏览器访问SDK的lib目录地址，和 angular.json的assets配置相关，参考 Basic Setup。

# 2. 模块化导入 Addon

  1. Install

    npm install -D gulp @foxitsoftware/gulp-merge-addon

  2. 编写合并 addon 的 gulp 任务， 参考 </integrations/angular/gulpfile.js>

  3. 修改 package.json 的 scripts 配置， 如下：

    "scripts": {
    "postinstall": "npm run update-sdk",
    "update-sdk": "node bin/setup.js",
    "merge-addons": "gulp merge-addons",
    "start": "npm run merge-addons && ng serve",
    "build": "npm run merge-addons && ng build",
    "test": "npm run merge-addons && ng test",
    "lint": "set NODE_OPTIONS=--max-old-space-size=8192 && ng lint",
    "e2e": "ng e2e"
},

    这样每次运行npm start启动服务前会自动合并插件。

  4. 引用方法请参考 /integrations/angular/src/app/pdfviewer/pdfviewer.component.ts;

# 3. 引用 allInOne.js

在foxit-lib/uix-addons/目录下有一个已经合并好的allInOne.js，在pdfviewer.component.ts中按下面的方法导入js：

// ...
import * as UIExtension from 'path/to/foxit-lib/UIExtension.full.js';
import * as Addons from 'path/to/foxit-lib/uix-addons/allInOne.js';
// ...

在 onInit 方法构造 PDFUI 实例时这样传参给PDFUI:

this.pdfui = new UIExtension.PDFUI({
    addons: Addons,
    // other options
});

# 三种方式对比

引用方式 配置 增加的浏览器请求 可修改(国际化词条,addon.info.json中的配置==)
碎片化 无 n+ 支持修改
模块化 要配置gulp 0 支持修改，修改后需重新合并
allInOne 无 一个 不支持修改

# 运行您的应用程序

在命令行终端，运行：

npm start

太棒了，一切准备就绪。在浏览器中，输入 <http://localhost:4200> 来加载您的应用程序。
